Javascript Analogy!

HTML is what I consider to be the structure of when you’re building a game. CSS is the designs, models, etc, basically CSS is the thing that makes all the world environments in the game so beautiful.
And Javascript is the mechanics, Javascript is what makes the game work, without Javascript, the game wouldn’t be a game, and just some glitchy mess instead.

Control Flow & Loops!

Control flow is what is used to make code run in the order you want. It’s like telling someone to buckle your seatbelt before you drive your car.
As for loops, it’s like a routine, you keep doing that thing over and over again, you can add and remove certain things from your routine but you keep
looping through your routine until you don’t see any point in doing so anymore.

DOMS!

Web documents have a programming interface called the Document Object Model (DOM). So that software can alter the document's structure, appearance, and content, it serves as a representation of the page.
Programming languages can communicate with a page by interacting with the nodes and objects that the DOM uses to represent the document.

Arrays & Objects!

An array is a group of identically data-typed elements kept in consecutive memory regions. It’s kind of like a recipe or shopping list.
An object is almost like the heart of your code, it’s data that contains practically any and all variables. Without an object, your variables will be lost.

Functions!

A procedure must accept some input and return an output with some evident relationship between the input and the output in order to qualify as a function.
A function is a group of statements that performs a task or calculates a value.
